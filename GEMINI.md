# Botla-co Project Context

## Project Overview

Botla-co is a full-stack chatbot platform designed to provide AI-powered chat capabilities for websites. It features a robust backend for managing chatbots, processing data (RAG), and handling authentications, alongside a modern frontend dashboard and an embeddable chat widget.

### Tech Stack

- **Backend:** Go (1.25.4+), PostgreSQL, Redis, Qdrant (Vector DB), AWS S3 / Cloudflare R2.
- **Frontend:** React 19, Vite, TailwindCSS, Radix UI.
- **Widget:** Preact, Vite, TailwindCSS (Embeddable).
- **Infrastructure:** Docker Compose for local development.

## Architecture

The project follows a standard Go project layout with frontend applications in subdirectories.

- **`cmd/server/`**: Backend entry point (`main.go`).
- **`internal/`**: Private application logic (handlers, services, DB access).
  - **`api/`**: HTTP handlers and routing.
  - **`db/`**: Database queries (generated by `sqlc`).
  - **`rag/`**: RAG implementation (OpenAI, Qdrant).
  - **`scraper/`**: Web scraper for ingesting content.
  - **`services/`**: Core business logic.
- **`pkg/`**: Public shared code (config, logging, middleware).
- **`frontend/`**: The main dashboard application (React/Vite).
- **`widget/`**: The embeddable chat widget (Preact/Vite).
- **`db/migrations/`**: SQL migration files.

## Development Workflow

### Prerequisites

- Go 1.25.4+
- Node.js & npm
- Docker & Docker Compose
- `migrate` CLI tool (for database migrations)

### Key Commands

**Infrastructure & Database:**

```bash
make up               # Start PostgreSQL and Redis via Docker
make down             # Stop Docker services
make migrate-up       # Apply database migrations
make migrate-down     # Rollback migrations
make psql             # Connect to the database CLI
```

**Backend:**

```bash
make be-run           # Run the backend server (requires CGO/Fitz for PDF)
make be-run-no-pdf    # Run the backend without PDF support
make build            # Build the backend
make test-all         # Run all tests (including integration)
make lint             # Run linters (golangci-lint)
```

**Frontend:**

```bash
make fe-run           # Start the frontend dev server (runs 'npm run dev' in frontend/)
```

**Widget:**

Go to `widget/` directory to manage the widget:
```bash
cd widget
npm install
npm run dev
```

### Environment Variables

Configuration is handled via `.env` files. See `.env.example` in the root and `frontend/.env.example`.

**Critical Variables:**
- `DATABASE_URL`: PostgreSQL connection string.
- `REDIS_URL`: Redis connection string.
- `JWT_SECRET`: Secret for signing JWTs.
- `OPENAI_API_KEY`: Key for LLM operations.
- `QDRANT_URL`: Vector database URL.

## Coding Conventions

### Backend (Go)

- **Structure:** Follows Standard Go Project Layout (`cmd`, `internal`, `pkg`).
- **Database:** Raw SQL queries via `sqlc` (in `internal/db`). Avoid ORMs.
- **Error Handling:** Wrap errors with context using `fmt.Errorf("...: %w", err)`.
- **Testing:**
  - Unit tests alongside code (`*_unit_test.go`).
  - Integration tests in `internal/integration/`.
  - Prefer table-driven tests.
- **Logging:** Use `pkg/logger` for structured JSON logging.

### Frontend (React)

- **Styling:** TailwindCSS for utility-first styling.
- **Components:** Radix UI primitives for accessible components.
- **State Management:** React Query (`@tanstack/react-query`) for server state.
- **Testing:** Vitest for unit tests, Playwright for E2E.

## Testing Strategy

- **Backend:** High coverage requirement (>= 90%). CI runs `make ci` which includes `vet`, `lint`, and `test-no-pdf`.
- **Frontend:** Unit tests with Vitest, E2E tests with Playwright.
